DAO – Developers

Purpose: provide an inspectable coordination space for proposing changes to schemas, evidence contracts, deterministic view generation, and governance interfaces — anchored to published artefacts.

In scope

  • Questions about how a rendered view is produced from authoritative inputs.
  • Proposals to adjust schemas, contracts, or view structure (naming, fields, layout, traceability).
  • Proposals to improve reproducibility and version-to-view mapping.
  • Proposals to add new inspectable pages that reference existing published artefacts.

Out of scope

  • Uploading artefacts, datasets, or binaries through this portal.
  • Executing code, pipelines, or running any logic server-side.
  • Mutating authoritative state from this interface.

DAO proposals output format (summary)

  • File-level changes
  • Schema changes (if needed)
  • Tests / validation
  • Determinism guarantees
  • Regeneration steps

DAO template (Developers)

  • File-level change list
  • Deterministic regeneration requirement
  • Gate: scripts/run_example_report_clean.sh must pass in eudr-dmi-gil

This interface does not accept uploads and does not execute logic. Proposals must be grounded in published artefacts (for example: a rendered HTML view, a referenced schema, or a versioned report).
