DAO – Stakeholders

Purpose: provide a transparent forum for reviewing published artefacts, raising questions, and proposing governance-oriented changes to how evidence and system state are represented in this Digital Twin.

In scope

  • Questions about published views, evidence summaries, and governance records.
  • Proposals to clarify terminology, roles, responsibilities, and review processes.
  • Proposals to add or refine portal views that improve inspection and traceability.
  • Requests to link proposals to specific published artefacts and versioned references.

Out of scope

  • Uploading new datasets or evidence.
  • Running computations, pipelines, or any execution logic in this portal.
  • Changing the authoritative system directly from this interface.

DAO proposals output format (summary)

  • Questions raised
  • Acceptance criteria decisions (pass/fail thresholds, evidence sufficiency)
  • Developer follow-ups: file-level changes, tests, determinism, regeneration

DAO template (Stakeholders)

  • Observed gap
  • Evidence (portal URLs)
  • Requested change (human language)

This interface does not accept uploads and does not execute logic. Proposals must be grounded in published artefacts (for example: a specific rendered view, a referenced schema, or a versioned record).
